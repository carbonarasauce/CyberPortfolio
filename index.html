CASE-003 Network Forensics

Network Traffic & Packet Analysis

Built the habit of reading raw network traffic instead of trusting summaries — because most SOC tools eventually point back to "go look at the packets."

See Details ▾
Tools Used
Wireshark PCAP Analysis Protocol Filters TCP Stream Reconstruction
What I Did
📡

Captured and analyzed live and saved traffic, reading the protocol hierarchy before drilling into individual frames.

🔎

Used display filters to isolate specific conversations — unexpected ports, plaintext credentials, repeated attempts.

🔗

Followed TCP streams end-to-end to reconstruct what happened in a session, not just that something happened.

📈

Compared baseline "normal" traffic against anomalous captures to build instinct for what should draw attention.

Packet Analysis Wireshark Filters Traffic Baselining Anomaly Identification
CASE-004 Adversary Intelligence

Honeypot Deployment & Threat Intelligence

Got an unfiltered look at what automated attackers actually do — by giving them something to attack inside a safely isolated home lab.

See Details ▾
Tools Used
Kali Linux [Honeypot tool] Network Segmentation Log Analysis
What I Did
🏗️

Built and isolated a home lab network segment so the honeypot could be exposed without putting other devices at risk.

🍯

Deployed a honeypot on Kali Linux to passively log connection attempts, credentials tried, and commands run.

📋

Reviewed logs for patterns — common usernames and passwords attempted, scanning behavior, and follow-on actions.

🔗

Cross-referenced honeypot activity with Wireshark captures to connect app-layer logs with wire-level traffic.
